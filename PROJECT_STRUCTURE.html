🏗️ Application Architecture Overview

MVC Architecture

REQUEST → ROUTES → MIDDLEWARE → CONTROLLER → MODEL → DATABASE ↓ EVENTS (Broadcasting) ↓ WebSocket Broadcast → JavaScript Listeners → UI Update

Real-time Communication Flow

1. User sends message via UI 2. JavaScript sends HTTP POST to Controller 3. Controller saves to Database 4. Event is fired (MessageSent) 5. Event is broadcast to WebSocket channel (chat.{id}) 6. Laravel Echo receives broadcast 7. JavaScript listeners update UI in real-time 8. All connected users see message instantly

File Upload Flow (TUS Protocol)

1. User selects file 2. JavaScript (resumable_upload.js) initiates TUS upload 3. File chunks sent to TusController 4. TusController manages upload session 5. Completed file stored in public/uploads 6. Media model created with file metadata 7. Message sent with media attachment 8. Media displayed in chat UI

Authentication Flow

1. User submits login form (loginauth.js) 2. AuthController validates credentials 3. Sanctum generates API token 4. Session created with middleware checks: - EnsureAuthenticated - EnsurePhoneEntered - EnsureOtpVerified - EnsureProfileCompleted 5. User redirected to dashboard 6. WebSocket connection authenticated 7. Real-time features enabled

Key Dependencies Relationship

Chat Model ├── Has Many Messages ├── Has Many Participants (ChatParticipant) └── Relates to Multiple Users Message Model ├── Belongs To Chat ├── Belongs To User (sender) ├── Has Many Media attachments ├── Can have PinnedMessage record ├── Can have StarredMessage record └── Can be a reply (reply_to field) User Model ├── Has Many Chats (through ChatParticipant) ├── Has Many Messages ├── Has Many UserBlocks ├── Last seen timestamp ├── OTP Verification status └── Device & Session records

🚀 Getting Started / Setup Instructions

Prerequisites

PHP 8.2+
Server-side language
Node.js & NPM
Front-end build tools
Composer
PHP package manager
Database (MySQL/PostgreSQL)
Data persistence

Installation Steps

1. Clone repository git clone [repo-url] cd chatapp 2. Install PHP dependencies composer install 3. Setup environment cp .env.example .env php artisan key:generate 4. Configure database in .env DB_HOST=localhost DB_DATABASE=chatapp DB_USERNAME=root DB_PASSWORD= 5. Run migrations php artisan migrate 6. Install NPM dependencies npm install 7. Build front-end assets npm run build 8. Start development server php artisan serve 9. In another terminal, start WebSocket server php artisan reverb:start 10. In another terminal, start queue listener php artisan queue:listen 11. Access application http://localhost:8000

Development Commands

composer dev # Run all dev services concurrently npm run dev # Watch and rebuild assets php artisan migrate # Run database migrations php artisan test # Run tests php artisan tinker # Interactive shell php artisan queue:work # Process background jobs
